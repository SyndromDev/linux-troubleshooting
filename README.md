# Linux troubleshooting

## Что делал
Практиковался в диагностике проблем с Linux-сервисами и сетью.

## Кейс: nginx не запускается
Проверял статус сервиса:
systemctl status nginx

Смотрел ошибки:
journalctl -u nginx -xe

## Кейс: порт занят
Проверял занятые порты:
ss -tulnp | grep :80

Определял процесс:
lsof -i :80

## Кейс: нет доступа к сайту
Проверял сеть:
ping google.com

Проверял сервис:
curl localhost

## Подход к диагностике
Сначала проверяю сервис → затем порт → затем логи → затем сеть.

## Что понял
— как находить причину, если сервис не работает  
— как проверять сеть и доступность  
— как выявлять конфликты портов
